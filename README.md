# 1
Так как нам необходимо делать вычисления на ЦПУ, далее сохранять на диск и передавать по сети, то можно выделить соответствующие метрики:
- CPU LA - информация о загрузке ЦПУ;
- IOwait - указание на то, сколько процессору нужно ждать завершиния операций с диском или сетью;
- IOPS - ввод/вывод с(на) диск, количество операций в секунду;
- inodes - дескрипторы файлов, если все заняты, есть вероятность получить ошибку при создании файла;
- FS - наблюдение за свободным местом на диске;
- NetTaffic - пропускная способность сети.

# 5

Push модель (pros):
  - упрощение репликации данных в разные системы мониторинга или их резервные копии;
  - более гибкая настройка отправки пакетов данных с метриками;
  - UDP — это менее затратный способ передачи данных, из-за чего может возрасти
производительность сбора метрик;

Pull модель (pros):
  - легче контролировать подлинность данных;
  - можно настроить единый proxy server до всех агентов с TLS;
  - упрощённая отладка получения данных с агентов;

Push модель (cons):
  - высокая вероятность ложных срабатываний;
  - зависимость от качества настроек;
  - сложность в управлении из-за большой нагрузки и потенциальной необходимости в ручной настройке.
